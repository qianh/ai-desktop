# AppScope Context

## Glossary

### Chrome Session MVP

A product slice where AppScope captures traffic only from Chrome instances that AppScope launches with an isolated profile and explicit proxy settings.

It includes Add Page, Chrome launch, local proxy startup, CA readiness, persisted real flows, flow list/detail UI, search/filter, and minimal HAR/JSON export.

It excludes arbitrary macOS app transparent capture, Network Extension capture, Safari/system proxy capture, SSL pinning bypass, cloud sync, and the 10k-flow performance target.

### Proxy Sidecar

The mitmproxy/mitmdump runtime that AppScope starts and stops for a capture session.

The product should prefer an embedded Tauri sidecar binary. During development, using `mitmdump` from the local `PATH` is acceptable when the UI and diagnostics clearly report that fallback. Simulated flow events are not a valid substitute for real captured proxy traffic.

### AppScope Local CA

A local Root CA certificate generated by AppScope or its proxy runtime on the user's Mac for decrypting HTTPS traffic in user-started capture sessions.

The user does not provide this certificate. AppScope creates it locally, stores it under the AppScope application support directory, guides the user to trust it in macOS Keychain, checks trust status, and can remove it. Automated MVP verification may use HTTP to avoid requiring admin trust prompts, while HTTPS is manually accepted only after the local CA is trusted.

### Flow Store

The local SQLite persistence layer for pages, apps, sessions, and captured flows.

For the Chrome Session MVP, flow list/detail and exports must read from SQLite-backed real captured data. Request and response bodies only need preview text and metadata in this slice; full large-body file storage is reserved behind the same directory and API boundary.

### Session Export

A per-session export generated from the Flow Store.

The Chrome Session MVP includes JSON export and a minimal HAR 1.2 export for the current session. Sensitive headers and cookies are masked by default. Multi-session batch export, cURL export, replay, map local, and rewrite are outside this slice.

### App Entry

A saved local macOS application launcher entry.

For the Chrome Session MVP, Add App means scanning installed apps, saving an app entry, and launching the app normally. System Proxy Capture and Transparent Capture are not accepted behavior for this slice, and App Entries are not expected to produce captured flows.

### App Chat

An in-app conversational workspace owned by AppScope itself, with local chat history, model selection, and memory. It is distinct from the built-in external Chat page that AppScope launches or captures as a Page.
_Avoid_: using "Chat" alone when the distinction from the external Chat page matters.

### Model Provider

A configured backend that can answer an App Chat message. Deepseek and GLM are API providers; ChatGPT is represented by the local Codex CLI provider in the first App Chat slice.
_Avoid_: assuming every provider is a CLI command or every provider is a direct cloud API.

### Global Memory

A local collection of user-editable facts that every App Chat conversation may use as context. It is shared across App Chat threads and is separate from per-thread message history.
_Avoid_: Conversation summary, browser cache, captured traffic.
